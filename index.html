Reducing Information Overload Through Adaptive Forgetting

This project is a note-centric web browser that acts as an intelligent workspace: instead of forcing users to manage too many tabs and notes, it dynamically decides what should stay active, fade, archive, or be removed, which aims to help users protect limited working memory.

The Problem

Digital workspaces overload working memory

Modern knowledge work happens in environments full of tabs, notes, pages, and interruptions. But attention and working memory are limited, so when too many items compete for focus, users experience higher mental effort, slower completion, and more mistakes.

📚

Too Much Information

Research, planning, and studying now happen across many digital sources at once, making overload routine.

🧩

Limited Working Memory

Only a small amount of information can stay cognitively active, so too many open items quickly become costly.

⚠️

Current Tools Miss the Gap

Most tools optimize remembering, but very few help users selectively forget irrelevant information at the right time.

The Core Idea

An intelligent browser that manages memory like a cognitive system

Instead of treating every tab or note equally, the interface organizes items into a working set for active information and a library for archived information. The system then updates priority over time using memory decay and reinforcement learning.

🗂️

Working Set

Contains the items needed right now—like a digital version of working memory.

🏛️

Library

Stores archived information that remains available but no longer competes for attention.

📉

Adaptive Decay

Each item loses strength over time unless reinforced by user interaction.

🤖

RL Personalization

The agent learns from user behavior to improve future keep/fade/archive/trim decisions.

How It Works

From interaction to intelligent forgetting

Each item has attributes such as type, state, strength, access history, and pinned status. These features help the system estimate relevance and decide how that item should be managed.

01

Track Item Features

Tabs, notes, and pages are represented as structured items with features like last access time, strength, tags, and interaction history.

02

Apply Memory Decay

Inspired by the forgetting curve, strength decays exponentially when an item is not used. Reopening, editing, searching, or scrolling boosts it again.

03

Choose an Operation

Based on updated strength and context, the system decides whether the item should stay active, fade in priority, be archived to the library, or be trimmed with a restore option.

04

Learn From Feedback

If users reopen a trimmed item, that counts as negative feedback. If system decisions match user behavior, that becomes positive reinforcement for future predictions.

Evaluation

How success is measured

The system is evaluated by comparing three conditions: a standard browser, a static organization tool, and the full adaptive forgetting system. The goal is to reduce working memory burden while preserving task performance.

Condition Description Purpose
Standard Browser No cognitive support or adaptive organization. Baseline for overload.
Static Organization Tool Manual organization without machine learning. Tests whether organization alone helps.
Full System Adaptive forgetting plus reinforcement learning. Tests the full intelligent intervention.
🧠

Cognitive Load

Measured with NASA-TLX, focusing especially on mental demand, effort, and frustration.

🔁

Behavioral Indicators

Includes number of active tabs, frequency of tab switching, and task completion time.

📌

How closely system decisions match real user preferences over time.

Impact

The key impacts

This project shifts the focus from “How can technology help us remember more?” to “How can technology help us manage attention better?” The result is a browser that supports cognitive efficiency by deciding what deserves to stay in mind.

Most productivity systems focus on storage and retrieval. This design treats forgetting as a useful cognitive function and turns it into an adaptive interface strategy.

It allows the system to personalize memory management instead of relying on one rigid rule. Over time, the browser learns what this user tends to revisit, ignore, or protect.
